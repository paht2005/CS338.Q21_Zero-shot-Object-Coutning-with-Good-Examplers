# Integrations — Zero-shot Object Counting with Good Exemplars

## External APIs

### Google Gemini API
- **Used by:** `code/source-code/prompt_enhancer.py`, `demo/prompt_enhancer.py`
- **Purpose:** Generate "Rich Prompts" — visual definitions of target objects for better detection
- **Model:** `gemini-2.0-flash-exp` (configurable via `GEMINI_MODEL_NAME` env var)
- **Auth:** `GEMINI_API_KEY` environment variable (loaded from `.env` via python-dotenv)
- **Calls:** `model.generate_content([prompt, image])` — multimodal (text + image) input
- **Fallback:** `enhance_prompt_simple()` — template-based prompt without API call
- **Error handling:** Retries up to `max_retries=2`, falls back to original class name on failure

### Weights & Biases (wandb)
- **Used by:** `code/source-code/FSC_train.py`, `code/source-code/FSC_pretrain.py`
- **Purpose:** Experiment tracking — loss curves, metrics, hyperparameters
- **Auth:** `WANDB_API_KEY` environment variable (optional)
- **Project:** Configurable via `--wandb` arg (default: `"counting"`)
- **Archived runs:** `experiments/exp{2,3,4,5}/wandb/`

### OpenAI CLIP (via GitHub)
- **Used by:** Multiple files for image-text similarity scoring
- **Installed:** `pip install git+https://github.com/openai/CLIP.git`
- **Models used:** `ViT-B/32` (binary classifier), `ViT-L/14` (scoring in GroundingDINO pipeline)
- **No API key required** — runs locally

## Pre-trained Model Downloads

| Model | Source | How Loaded |
|-------|--------|------------|
| CLIP ViT-B/32, ViT-L/14 | OpenAI (auto-downloaded by `clip.load()`) | `clip.load("ViT-B/32", device)` |
| YOLO-World v2 | Ultralytics (local `.pt` file) | `YOLOWorld("./yolov8x-worldv2.pt")` |
| GroundingDINO | Vendored code + local weights | `load_model(config_path, weights_path)` |
| MAE ViT-Base | Custom checkpoint | `torch.load(checkpoint_path)` |

## Dataset

### FSC-147
- **Source:** Kaggle (`https://www.kaggle.com/datasets/xuncngng/fsc147-0`)
- **Location:** `code/source-code/data/FSC147/`
- **Not in git** — only a small sample tracked in `data/FSC147/sample/`
- **Components:** Images (384×384), density maps, annotations (JSON), train/val/test splits, class labels

## File-based Integrations

- **Annotation JSONs** — exemplar bounding boxes produced by detection scripts:
  - `annotation_FSC147_384.json` (baseline)
  - `annotation_FSC147_pos.json`, `annotation_FSC147_neg.json` (GroundingDINO)
  - `annotation_FSC147_pos_prompt.json`, `annotation_FSC147_neg_prompt.json` (GroundingDINO + Rich Prompt)
  - `annotation_FSC147_pos_yolo_prompt.json` (YOLO-World + Rich Prompt)
- **Rich Prompt text files** — per-image prompts generated by Gemini:
  - `pos_prompt_enhanced.txt`, `neg_prompt_enhanced.txt`
- **Density map `.npy` files** — ground truth density maps per image
- **Checkpoint `.pth` files** — model weights for different training variants

## No Database / No Backend Server

This is a research project — no database, no REST API, no message queues. All data flows through:
1. Local files (images, JSON annotations, text prompts, `.npy` density maps)
2. PyTorch tensors in memory
3. Streamlit for demo UI (runs as a local web server)
