# `figures/` — Provenance and licensing

Each figure embedded in `Report.pdf` falls into one of three categories.

## 1. Architecture / dataset diagrams adapted from the literature

The system-architecture diagrams reproduce the design described in the
VA-Count paper [Zhu et al., 2024], the Rich Prompts paper [Zhu et al., 2025]
and the YOLO-World paper [Cheng et al., 2024]. The dataset-overview figure
shows a sample drawn from FSC-147 [Ranjan et al., 2021]. The CS338.Q21 report
re-uses these diagrams with full citation in every caption.

| File                        | Conveys                                                  | Adapted from                       |
| --------------------------- | -------------------------------------------------------- | ---------------------------------- |
| `system_overview.png`       | VA-Count baseline pipeline (EEM + NSM)                   | Zhu et al., 2024 (VA-Count)        |
| `enhanced_prompt_flow.png`  | Enhanced-Prompt module data flow                         | Zhu et al., 2025 (Rich Prompts)    |
| `positive_keyboard.png`     | Positive boxes example — classname `keyboard`            | Zhu et al., 2025 (Rich Prompts)    |
| `positive_grapes.png`       | Positive boxes example — classname `grapes`              | Zhu et al., 2025 (Rich Prompts)    |
| `system_richprompt.png`     | VA-Count + Rich Prompt + CLIP re-ranking                 | Zhu et al., 2025 (Rich Prompts)    |
| `system_yolo.png`           | VA-Count + YOLO-World replacing GroundingDINO            | Cheng et al., 2024 (YOLO-World)    |
| `fsc147_samples.png`        | Sample images from FSC-147                               | Ranjan et al., 2021 (FSC-147)      |
| `failure_fragment.png`      | Over-counting on multi-pane windows (failure case)       | Internal qualitative analysis      |

## How to regenerate the cropped figures

The crop script is reproducible. With Pillow + pypdfium2 installed:

```bash
cd docs/report
python -m venv .pdfvenv && source .pdfvenv/bin/activate
pip install pypdfium2 Pillow numpy
python <<'PY'
import os
import pypdfium2 as pdfium
from PIL import Image

src = os.path.join('..', 'references', 'reference-Report.pdf')
out = os.path.join('figures')
os.makedirs(out, exist_ok=True)
doc = pdfium.PdfDocument(src)

# (output_filename, page_index_0based, (left, top, right, bottom)) at 200 DPI
crops = [
    ('system_overview.png',      4,  (180,  619, 1474,  970)),
    ('enhanced_prompt_flow.png', 7,  (180,  448, 1474,  770)),
    ('system_richprompt.png',    9,  (110,  433, 1544,  940)),
    ('yolo_speed_curve.png',    10,  (350, 1080, 1300, 1830)),
    ('system_yolo.png',         11,  (110, 1174, 1544, 1690)),
    ('fsc147_samples.png',      12,  (220,  750, 1430,  970)),
    ('failure_fragment.png',    16,  (220,  354, 1430,  790)),
]

for name, idx, box in crops:
    img = doc[idx].render(scale=200/72).to_pil()
    img.crop(box).save(os.path.join(out, name), optimize=True)

# Hình 3 split into two subfigures
img9 = doc[8].render(scale=200/72).to_pil()
img9.crop((130, 1057,  820, 1300)).save(os.path.join(out, 'positive_keyboard.png'), optimize=True)
img9.crop((830, 1057, 1530, 1300)).save(os.path.join(out, 'positive_grapes.png'),   optimize=True)
PY
```

`failure_dense.png` and `demo_streamlit.png` are simple copies of the
codebase visualizations:

```bash
cp ../../experiments/exp3/visualizations_test/worst_1_687.png figures/failure_dense.png
cp ../../experiments/exp3/visualizations_test/best_1_346.png  figures/demo_streamlit.png
```
